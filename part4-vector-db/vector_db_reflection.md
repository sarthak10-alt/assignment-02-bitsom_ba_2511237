# # Vector DB Use Case Reflection

**Would a traditional keyword search suffice?**
No. A keyword search only looks for exact words. If a lawyer searches for "termination clauses," a keyword search might miss a paragraph that uses the words "ending the contract" or "agreement cancellation."

**Why use a Vector Database?**
A Vector Database captures **semantic meaning**. It understands that "termination" and "ending a contract" are the same thing. 

**The Role of Vector DB in this system:**
1. **Contextual Search:** It allows lawyers to ask questions in plain English.
2. **Efficiency:** It can scan 500-page documents instantly to find relevant sections based on the *concept* of the question, not just the words.
3. **Accuracy:** It reduces the time spent manually reading through pages that aren't relevant.
